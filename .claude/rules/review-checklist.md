# coding-vibe 代码审查清单

## 自动拦截项（CI 应覆盖）

- [ ] 代码能编译/类型检查通过
- [ ] lint 无报错
- [ ] 测试全部通过
- [ ] 测试覆盖率不降低
- [ ] 无已知安全漏洞依赖

## 功能正确性

- [ ] 实现是否满足需求？边界条件有测试？
- [ ] 错误路径是否处理（网络超时、空响应、非法输入）？
- [ ] 并发问题（竞态、死锁、一致性问题）？
- [ ] 是否有硬编码值（IP、端口、密钥、URL）？
- [ ] 时间/时区处理是否正确？

## 安全性

- [ ] 输入验证（XSS、SQL 注入、命令注入、路径遍历）？
- [ ] 认证/授权检查（用户能否越权访问）？
- [ ] 敏感信息是否泄露（日志、错误消息、响应体）？
- [ ] 速率限制 / 防暴力破解？
- [ ] 依赖是否有已知漏洞？

## 性能

- [ ] N+1 查询？是否用了批量查询？
- [ ] 不必要的重复计算？需要缓存？
- [ ] 大对象/大列表是否有分页/流式处理？
- [ ] 资源（连接池、文件句柄）是否正确释放？
- [ ] 是否有 O(N²) 或更差的算法可优化？

## 可维护性

- [ ] 命名清晰（变量名说明意图而非类型）？
- [ ] 函数单一职责（≤50 行）？
- [ ] 文件大小适中（≤800 行）？
- [ ] 注释解释 WHY 而非 WHAT？
- [ ] 是否有复制粘贴的重复代码？
- [ ] 配置是否抽离（环境变量 / 配置文件）？
- [ ] 错误消息是否对用户友好、对开发者可 debug？

## 测试质量

- [ ] 新代码是否有测试？
- [ ] 测试命名清晰（`test_<function>_<scenario>_<expected>`）？
- [ ] 测试是行为验证而非实现细节？
- [ ] 是否有边界条件测试（空、极限值、重复）？
- [ ] 异常路径测试覆盖率？

## 可观测性

- [ ] 关键操作有日志（INFO 级）？
- [ ] 错误有详细上下文日志（ERROR 级）？
- [ ] 关键指标有监控（请求量、延迟、错误率）？
- [ ] 是否有健康检查端点？

## 兼容性

- [ ] API 变更是否向后兼容？
- [ ] 数据库迁移是否可回滚？
- [ ] 配置变更是否兼容旧版本？
- [ ] 删除/废弃的接口是否有过渡期？(deprecation warning)

## Review 输出格式

```markdown
## 文件: <path>

### 严重问题 🚨
- <问题描述>（建议方案）

### 建议 💡
- <优化建议>（理由）

### 问题 ❓
- <需要作者澄清的地方>
```
